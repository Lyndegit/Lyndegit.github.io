解决打不开 xxx.github.io的方法

由于DNS的某方面问题，github.io后缀的网站经常崩，可以说让人非常心累，那么有什么可以一举多得配置一次，近期内多次访问都不会报404的方法呢？

网上的方法都是采用修改 hosts文件的方法，将网站的地址添加到host文件中，路径：C:\Windows\System32\drivers\etc 并手动添加ip地址的域名映射。

这样固然是一个较为妥当的解决方法，但不能解决所有github的pages打不开的问题，所以我选择修改DNS地址，虽然host请求级别高于dns，但是我最近经常访问github，别人的博客，或者项目文档都是xxx.github.io的地址，一个一个查服务器地址再添加host，太繁琐了。

以win10为例，打开 控制面板> 网络与Internet设置 > 网络共享中心
点击WLAN连接，在WLAN状态中点击->属性->Internet 协议版本4 (TCP/Ipv4),修改dns地址，将备用dns地址设为 8.8.8.8 (谷歌）首选可以设为本地运营商的或者公共dns服务器的，百度都可以查到；因为谷歌的服务器是在国外，所以可以访问形如xxx.github.io的网站，但是访问国内的网站速度可能会慢，所以才设为备选；首选设置成国内的，这样不影响正常使用。这里我首选服务器配置的是阿里的，备用选择谷歌。
