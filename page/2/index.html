浅谈摄影

对于摄影：情不知所起，一往情深

第一次接触摄影，是当时大一时被学生会摄影部的同学拉去当模特，拍完之后，虽说看到帅气的照片后非常开心，但是心里依然非常好奇，似乎对于我来说，比起被当成模特，我更愿意手持机器，看着目镜通过按下快门那一瞬间的喀嚓声获得更大的快感。于是，说干就干，从基本的单反入手，到胶片定焦以及从摄影到摄像，我似乎越来越迷恋于拍摄的乐趣。是啊：关于人的记忆，往往是主观且会随着时间消逝的，唯有照片可以记录下这最美好的瞬间，让最好的时刻定格。

既然是浅谈摄影，这里来简单说一下摄影摄像的基础及这几年入门的心得吧。所谓摄影个人认为无非是3大块：目标素材，构图，光线。

俗话说的好：横看成岭侧成峰远近高低各不同，而摄影也是将3维物体对2维平面的投影，对一个目标物体来说摄影机的角度不同，其最后的呈像效果也会有所不同。并且我们知道，对于一个正常曝光的照片，他的主体（对焦点）前后都会有一个清晰的范围，超过这个范围后，景物会越来越模糊，这就是“景深”对于我们的目标素材，我们往往需要配合光圈和焦距物距等达到镜头与其的合理距离，使拍摄主体更为突出。并且有时我们使用慢门拍摄快速移动的物体（提高曝光时间）还会达到意想不到的效果（这是我拍过的一个车尾灯图片，提高曝光时间后：夜晚车的光源留下，而车竟然”消失”了）

对于构图来说：有n多种构图方式（对称构图，三分法，垂直水平线构图，曲线构图等等），其目的是摄影师通过某种形式的画面向别人传递所想要表达的特定信息与自己的主观意向，这就是构图的意义所在，一个好的摄影师会将所有元素和其布局组成考虑到一个有限的篇幅内，不同的点线面组成的物体，及光线和氛围的渲染成就一张良好的照片。（最近拍摄的一张人像，采用对称构图法，人物模特处于正中，通过不同的高光及阴影将画面一分为二，使被拍摄客体的五官更加立体）

光线：对于摄影师来说，巧妙使用光线是进阶课程，万千世界的各个物体，无不是反射色光后进入人眼或相机的CMOS中出现了形形色色的画面，光线又分为侧光，顺光逆光高逆光，柔光等等等等，这里不展开赘述。光具有较重要的表达性，有些光是硬的、刺目的、聚集的、直接的；有些光是软的、柔和的、散射的、间接的。在摄影中，光能影响被摄体再现的形状、影调、色彩、空间感以及美感、真实感。光线的三个特性是：强度、方向、色温。光的强度与光源的能量距离以及传播、设备介质有关。直观的感觉就是光的明暗程度。方向即是光源的方向。色温则表现在光源的颜色上。这三个特性会有助于我们在拍摄的时候更好分析光线，具体会在后面进行进一步更新。

关于21年欧冠的看法（一）

发表于 2021-02-18 更新于 2021-02-19 分类于 浅谈足球

我们正在经历一场足球时代的交接，新王诞生！

足球著名解说员詹俊

相信不少小伙伴都熬夜看了这场16强第一回合的关键比赛：巴萨罗那vs巴黎圣日耳曼。作为一个大巴黎死忠兼姆吹球迷，我当然也不能错过，赛前的的各种新闻以及网上的一些舆论让所有人都认为巴黎是处于劣势————毕竟巴黎方面：内马尔和迪玛利亚，两位前场不可或缺的球员伤停;而巴萨方面：联赛状态逐渐回温，皮克火线复出。
谁能想到这场比赛竟发展成一边倒的局面。
正所谓：
长江后浪推前浪，把前浪打在沙滩上。
老夫聊发少年狂，金球奖往事好辉煌。
可是今天谁承想，法国天才横空出世。
自古英雄出少年，让他小儿门口称王！
真：巴黎剩姆院，踩着梅球王完成了帽子戏法。
然而？我们来复盘这场比赛，究竟是姆皇太过耀眼，还是巴萨漏洞太多？鄙人来简单发表一下看法。
俗话说的好，足球是一项团队运动，我深信不疑，诚然：一个优秀的球员可以在最后10几码的区域完成很多工作，然而没有一个团队的支撑，一个人也是秀不起来的。
我们来看姆巴佩第一个进球：巴黎从后场发展到前场，看似球飞过了整个区域，其实总共才三次传球就把巴萨防线交的天翻地覆，由于帕雷德斯的长传转移和维拉蒂的穿插引线使得姆巴佩最终在小禁区舒服的拿到了球，最后这一拨一射，的确是巨星气质显现，其实也就比朗格莱快了半步，然而就是这半步，造成了进球。
同时姆巴佩的第三个进球可谓是快速反击的教科书式进球，巴萨阵地战进攻前压太深，而又不对姆巴佩设专人贴身防守，中场断球后，巴黎迅速形成3打2的局面，最终又是姆巴佩的速度以及射术，让比赛失去了悬念。此情此景又让人回想到18年世界杯，姆巴佩——那个风一样的少年，横空出世。用今天同样的方式，让世人记住了他的名字。
本人看球玩球也算有了几年，自己浅见:所谓现代足球，讲求快准狠，快速通过中场形成致命打击，这也是我为什么如此看好姆巴佩。诚然，比起意志力和盘带，姆巴佩比起同时期的C罗以及梅西差了不少，但是如今的战术体系中，像姆巴佩的球员可以说是教练最喜欢的，利物浦的渣叔讲究高位逼抢玩起来了“重金属”足球；包括穆里尼奥，也是大巴反击的重视拥护者。这也同样反应了如今的某些社会现象：讲求高效率，团队协作。因此就正如本文开头引用的“我们正经历一场足球时代的交接”，tikitaka对应防守反击，姆巴佩对应梅西，总有人正“年轻”。
以上扯淡居多^_^，总之还是希望今年大巴黎能走更远

浅谈前端vue.js的mvvm的框架

初次听到这个mvvm框架，我们似乎很难不联想到另一个java非常火爆的框架——MVC，其实，MVVM就是MVC的适配于vue的改进版本

其中： M：model；V:view; VM:ViewModel。为我们前端提供了一种将视图UI和数据业务逻辑分开的开发方法。MvvM的核心是ViewModel层，他就像是一个中转站，提供view（视图Ui）与Model（数据层）的双向数据绑定，该层向上与视图层进行双向数据绑定，向下与 Model 层通过接口请求进行数据交互，起呈上启下作用（数据驱动）。

图示如下：

Model层：是指数据模型，泛指后端进行的各种业务逻辑和数据操作，还包括向前段暴露的部分接口，主要围绕数据库系统展开。

ViewModel层：ViewModel 是由前端开发人员组织生成和维护的视图数据层。在这一层，前端开发者对从后端获取的 Model 数据进行转换处理，做二次封装，以生成符合 View 层使用预期的视图数据模型。由于实现了双向绑定，ViewModel 的内容会实时展现在 View 层，这是激动人心的，因为前端开发者再也不必低效又麻烦地通过操纵 DOM 去更新视图，MVVM 框架已经把最脏最累的一块做好了，我们开发者只需要处理和维护 ViewModel，更新数据视图就会自动得到相应更新，真正实现数据驱动开发。看到了吧，View 层展现的不是 Model 层的数据，而是 ViewModel 的数据，由 ViewModel 负责与 Model 层交互，这就完全解耦了 View 层和 Model 层，这个解耦是至关重要的，它是前后端分离方案实施的重要一环。

View层：前段大部分代码机器ui设计在此进行，这里需要向用户提供优美的前端页面和方便的人机交互接口。（这里推荐几个框架：包括Element，Ant Design of Vue等等等。）
