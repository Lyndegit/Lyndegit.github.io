进程池

名词解释：所谓进程池，顾名思义就是存储进程的“池子”。

在利用python进行系统管理的时候，特别是同时操作多个文件目录，或者远程控制多台主机开启多个进程，并行操作可节约大量的时间。当被操作对象数目不大时，可直接利用multiprocessing 中的 Process动态生成多个进程，但当进程数量成百上千时利用手动配置进程太过繁琐，可利用进程池。

Pool可以提供制定数量的进程，供用户调用，当有新的请求提交到pool中时，如果池还没有满，就会创建一个新的进程来执行请求，池中达到最大值时请求会等待阻塞在序列中的进程，直到进程结束才会创建新的进程（池子的逻辑结构类似于一个进程队列）

进程与线程

进程:是程序运行的过程，是系统进行资源分配和调度的一个独立单位

线程:是操作系统能够进行调度的最小单位。它被包含在进程之中，是进程中实际运作单位

关系:一个线程可以创建和撤销另一个线程；同一个进程中多个线程之间可以并发执行，相对进程而言，线程是一个更加接近于执行体的概念，他可以同其他线程共享数据，但拥有自己的栈空间，具有独立的执行序列

因此我们知道多个进程之间的数据是独立的（不共享），同一进程的线程共享数据（抢占cpu资源）

关于npm装包时-s和-d的区别

发表于 2021-02-04 更新于 2021-02-19 分类于 npm

-D后，安装包会在package中的 devDependencies对象中。简称dev。dev是在开发环境中要用到的。

-S后，安装包会在package中的 dependencies 对象中。简称dep。dep是在生产环境中要用到的。

举个例子：

构建工具：gulp和webpack是用来压缩代码，打包等需要的工具，程序实际运行的时候并不需要，就要放在dev中所以要用 -D

项目插件：例如element ui、echarts这种插件要在运行中使用的，就要放在dep中所以就用 -S

一般我们项目插件，在api中都可以看到，一般都是-S。因为这些插件是在程序运行中使用的。

一、什么是设计模式？
设计模式是面向对象语言特有的内容，是我们在面临某一类问题时候固定的做法。即：前人总结经验所做出的特定总结性解决方案
初学者最常用两个常用的模式：工厂模式，单例模式
二、工厂模式：创建者和调用者分离，使用专门的工厂类将选择实现类、创建对象进行统一的管理

三、单例模式：核心作用是确保一个类只有一个实例（避免反复调用反复生产，造成资源浪费）并且提供一个方芬该实例的全局访问点。
即:单例模式只生成一个实例对象，减少资源开销。当一个对象的产生需要比较多的资源，如读取配置文件，产生其他依赖对象时，可以产生一个“单例对象”，然后永久驻留内存中，从而极大降低开销。
